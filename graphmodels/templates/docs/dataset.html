Data file format

The data file need to be in a particular format for the System to work properly. Generally speaking, the data file must consist of multiple samples. In addition, a sample consists of multiple (but fixed-numbers) columns. Each sample must be represented in one line, with the columns separated by either white space, tabular, semicolon or comma. Each column represents the values of one variable or attribute. In addition, you could add a header to the top line of the data, with the same number of columns indicating the variable names.

You can give as many columns as you like, however the number of columns must be fixed through all samples. Furthermore, the data type of the sample values must be either integer or numeric.

Here's an example of such a data file describing attributes of abalones: 

length  diameter    height  whole_weight    shucked_weight  viscera_weight  shell_weight    rings
0.455   0.365       0.095   0.514           0.2245          0.101           0.15            15
0.35    0.265       0.09    0.2255          0.0995          0.0485          0.07            7
0.53    0.42        0.135   0.677           0.2565          0.1415          0.21            9
0.44    0.365       0.125   0.516           0.2155          0.114           0.155           10
0.33    0.255       0.08    0.205           0.0895          0.0395          0.055           7
0.425   0.3         0.095   0.3515          0.141           0.0775          0.12            8
0.53    0.415       0.15    0.7775          0.237           0.1415          0.33            20
0.545   0.425       0.125   0.768           0.294           0.1495          0.26            16
0.475   0.37        0.125   0.5095          0.2165          0.1125          0.165           9
...
There are 8 columns(variables) for each sample, indicating the Length, Diameter, Height and some other attributes of an abolone.

The header line can be ignored, In that case, the System will instead use V1,V2 ... to represent the variables.
